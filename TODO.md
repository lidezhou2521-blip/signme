# แผนการพัฒนาฟังก์ชันเพิ่มเติมสำหรับ SignMe (Feature Backlog)

เอกสารนี้รวบรวมรายการฟังก์ชันที่สามารถพัฒนาเพิ่มเติมเพื่อทำให้ระบบ SignMe สมบูรณ์และพร้อมสำหรับการใช้งานจริงระดับองค์กรมากยิ่งขึ้น

## รายการฟังก์ชันที่รอการพัฒนา (TODO)

- [x] **1. ระบบบันทึกประวัติเอกสาร (Audit Trail / Activity Log)**
  - บันทึกประวัติว่าเอกสารถูกสร้างเมื่อไหร่, ส่งหาใคร
  - บันทึกการเปิดอ่านเอกสาร (Viewed) พร้อมเก็บ IP Address และเวลา
  - แสดงผลเป็น Timeline ในหน้าแสดงรายละเอียดเอกสาร

- [x] **2. การสร้างไฟล์ PDF ฉบับสมบูรณ์ (PDF Flattening/Export)**
  - ใช้ไลบรารีของ PHP (เช่น mPDF, TCPDF, หรือ FPDI) เพื่อนำรูปลายเซ็นและพิกัด ไปฝังลงบนไฟล์ PDF ต้นฉบับ
  - ให้ผู้ใช้สามารถดาวน์โหลดไฟล์ที่ถูกเซ็นทับอย่างสมบูรณ์แล้วได้

- [ ] **3. ระบบแจ้งเตือนและทวงถาม (Reminders & Expiration Date)**
  - เพิ่มปุ่ม "ส่งอีเมลทวงถาม (Resend Email)" สำหรับเอกสารที่ยังรอเซ็น
  - ตั้ง Cron Job ส่งอีเมลเตือนอัตโนมัติหากผ่านไปหลายวันแล้วยังไม่มีการเซ็น
  - กำหนดวันหมดอายุ (Expiration Date) ของลิงก์เซ็นเอกสาร

- [ ] **4. ระบบจัดการเทมเพลต (Document Templates)**
  - บันทึกตำแหน่งกล่องลายเซ็นไว้เป็นเทมเพลตสำหรับเอกสารประเภทที่ใช้บ่อย (เช่น สัญญาจ้างงาน)
  - นำเทมเพลตมาใช้ซ้ำได้ทันทีโดยไม่ต้องลากวางใหม่ทุกครั้ง

- [x] **5. ฟิลด์รับข้อมูลแบบอื่นๆ (Additional Fields)**
  - เพิ่มประเภทของกล่องรับข้อมูลนอกเหนือจากลายเซ็น เช่น วันที่เซ็น (Date), ช่องกรอกข้อความ (Text input)
  - ผู้เตรียมเอกสารสามารถลากวางฟิลด์เหล่านี้ลงบน PDF ได้

- [ ] **6. โปรไฟล์ผู้ใช้งานและลายเซ็นที่บันทึกไว้ (User Profile & Saved Signatures)**
  - หน้าโปรไฟล์ส่วนตัว สำหรับผู้ใช้งานภายในระบบ (Internal Users)
  - ฟังก์ชันสร้าง/อัปโหลด และบันทึกลายเซ็นของตัวเองเก็บไว้ เพื่อเรียกใช้ได้รวดเร็วในครั้งถัดไป

- [ ] **7. หน้า Dashboard แบบสถิติ (Analytics Dashboard)**
  - สรุปตัวเลขสถิติและกราฟ เช่น จำนวนเอกสารทั้งหมด, จำนวนเอกสารที่รอเซ็น, และจำนวนเอกสารที่เซ็นเสร็จสมบูรณ์แล้ว
